# Telegram-бот мониторинга газа

Бот проверяет публичные цены поставщиков для фиксированного сценария и отправляет ежедневный отчёт в Telegram.

## Параметры

- Коммунарка, 13 км от МКАД
- газгольдер 5 000 л
- остаток 2%
- целевой уровень 85%
- доставка 4 150 л

Бот не оформляет заказов, не отправляет заявки и не передаёт поставщикам телефон.

## Что умеет текущая версия

- `/start` — включить ежедневный отчёт и показать меню;
- `/prices` — проверить цены сейчас;
- `/settings` — показать параметры расчёта;
- `/stop` — отключить ежедневную рассылку;
- кнопки под сообщениями;
- хранение подписчиков и истории цен в SQLite;
- сравнение новой цены с предыдущей;
- ежедневная отправка в заданное время;
- адаптер СК-Газ, читающий публичную цену без заполнения формы.

Остальные поставщики добавляются постепенно как отдельные адаптеры. Если цена недоступна автоматически, бот показывает это явно.

## Самый простой запуск

### 1. Создайте бота

Откройте в Telegram `@BotFather`, выполните `/newbot` и скопируйте полученный токен.

### 2. Установите Python 3.10 или новее

```bash
python -m venv .venv
```

Windows:

```bash
.venv\Scripts\activate
```

macOS/Linux:

```bash
source .venv/bin/activate
```

### 3. Установите зависимости

```bash
pip install -r requirements.txt
```

### 4. Укажите токен

Windows PowerShell:

```powershell
$env:TELEGRAM_BOT_TOKEN="ВАШ_ТОКЕН"
$env:REPORT_TIME="09:00"
python main.py
```

macOS/Linux:

```bash
export TELEGRAM_BOT_TOKEN="ВАШ_ТОКЕН"
export REPORT_TIME="09:00"
python main.py
```

### 5. Включите рассылку

Откройте созданного бота в Telegram и нажмите **Start**. Бот сам сохранит нужный `chat_id`.

## Настройки окружения

- `TELEGRAM_BOT_TOKEN` — обязательный токен;
- `REPORT_TIME` — ежедневное время отчёта, по умолчанию `09:00`;
- `BOT_TIMEZONE` — часовой пояс, по умолчанию `Europe/Moscow`;
- `LOG_LEVEL` — уровень журналирования.

## Проверка

```bash
pytest
```

## Постоянная работа

Для ежедневной рассылки процесс `python main.py` должен быть постоянно запущен. Подойдут домашний компьютер, Raspberry Pi или небольшой VPS. Для первого запуска рекомендуется обычный polling: он не требует домена, HTTPS или webhook.

# Круглосуточный запуск на Railway

Этот вариант не требует держать домашний компьютер включённым. Бот запускается на Railway как постоянный worker и работает через Telegram polling. Публичный домен и webhook не нужны.

## Что понадобится

- аккаунт GitHub;
- аккаунт Railway;
- токен, полученный у `@BotFather`;
- файлы этого проекта.

## 1. Загрузите проект в GitHub

1. Создайте на GitHub новый пустой репозиторий, например `gas-price-monitor`.
2. Распакуйте архив проекта.
3. Загрузите в репозиторий всё содержимое папки `gas-price-monitor`.

Файл `.env` с настоящим токеном загружать нельзя. Токен будет добавлен отдельно в Railway.

## 2. Создайте сервис Railway

1. В Railway нажмите **New Project**.
2. Выберите **Deploy from GitHub repo**.
3. Подключите GitHub и выберите репозиторий `gas-price-monitor`.
4. Railway прочитает `railway.json` и запустит команду:

```bash
python main.py
```

Публичный домен создавать не нужно: Telegram-бот получает сообщения через polling.

## 3. Добавьте защищённые переменные

В сервисе Railway откройте **Variables** и добавьте:

```text
TELEGRAM_BOT_TOKEN=токен_из_BotFather
REPORT_TIME=09:00
BOT_TIMEZONE=Europe/Moscow
DATABASE_PATH=/data/gas_monitor.db
LOG_LEVEL=INFO
```

Токен не нужно вставлять в исходный код или публиковать на GitHub.

## 4. Подключите постоянный диск

SQLite хранит подписчиков и историю цен. Чтобы эти данные не исчезли после нового развёртывания:

1. Откройте сервис в Railway.
2. Добавьте **Volume**.
3. Укажите путь монтирования:

```text
/data
```

Переменная `DATABASE_PATH` уже направляет базу в `/data/gas_monitor.db`.

## 5. Проверьте запуск

В разделе **Deployments** откройте журнал последнего развёртывания. При успешном запуске процесс продолжает работать и не завершается сразу.

Затем:

1. откройте своего бота в Telegram;
2. нажмите **Start** или отправьте `/start`;
3. нажмите **Проверить цены**;
4. убедитесь, что бот прислал отчёт.

## Обновление бота

При изменении файлов и отправке нового коммита в GitHub Railway автоматически развернёт свежую версию. Подписчики и история сохранятся на подключённом томе.

## Важные замечания

- Не запускайте одновременно две копии бота с одним токеном через polling: они будут мешать друг другу получать обновления.
- После размещения на Railway локальную копию `python main.py` следует остановить.
- Если токен случайно опубликован, отзовите его через `@BotFather` и создайте новый.
